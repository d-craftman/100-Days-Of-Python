# Day 31: Transitioning to Object-Oriented Programming (OOP)

## 🎯 Objectives
- Understand the difference between Procedural and Object-Oriented Programming.
- Define a `Project` Class as a blueprint for studio work.
- Initialize objects using the `__init__` method.

## 🛠️ The Build: The Project Blueprint
In Month 1, a project was just data in a dictionary. Today, we elevate. We are creating a **Project Class**. This allows us to bundle the data (name, budget) and the actions (updating status, calculating tax) into one neat package. This is how professional, enterprise-grade software is structured.

### Key Features:
- **The Class:** A master template for all @TheCraftroomStudios projects.
- **Attributes:** Defining specific properties like `client_name`, `base_budget`, and `deadline`.
- **The Constructor (`__init__`):** The engine that "births" a new project object every time we need one.

## 🚀 Technical Takeaway
OOP makes code reusable and scalable. Instead of passing dictionaries around, we interact with "Objects" that know how to manage themselves. This is the foundation for building complex apps.
